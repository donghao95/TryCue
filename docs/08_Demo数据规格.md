# 08 Demo 数据规格

## 1. 目标

本文档提供固定 Demo 内容，用于开发、调试、测试和 Prompt 评估。

为什么需要 Demo Data：

```text
1. 固定输入便于比较不同版本 Agent 行为。
2. 可以判断系统问题还是内容样本问题。
3. 可以快速验证点开、结束浏览、评论、收藏、质疑等行为是否自然。
4. 可以用于截图、演示和验收。
```

## 2. Demo 封面图片要求

Demo 数据必须包含真实封面图片文件，存放在 `assets/demo/` 目录下。Agent 通过多模态 AI 视觉能力直接分析封面图片。

必须准备的封面图片：

```text
assets/demo/cover_parenting_tips.png      -- 育儿避坑封面图
assets/demo/cover_renovation_tips.png     -- 装修避坑封面图
assets/demo/cover_general_recommendation.png -- 普通种草封面图
```

每个 Demo 的基本信息中使用 `coverImage` 字段指向对应的图片文件路径。

## 3. Demo 1：育儿避坑内容

### 3.1 基本信息

```json
{
  "title": "这 8 个宝宝用品千万别乱买，新手爸妈很容易踩坑",
  "coverImage": "assets/demo/cover_parenting_tips.png",
  "scale": "quick"
}
```

### 3.2 正文

```text
宝宝出生前我也跟风买了一堆东西，后来才发现很多真的用不上。

这篇只说我自己踩过的坑：

1. 太复杂的温奶器
功能很多，但实际每天用的就一两个。新手爸妈半夜根本没精力研究复杂操作。

2. 颜值很高但不好清洗的奶瓶
拍照好看不代表好用，尤其是边角多的款，洗起来真的崩溃。

3. 过早买很大的婴儿床
家里空间不够的话，很容易变成堆衣服的地方。

4. 各种安抚玩具一次性买太多
每个宝宝喜欢的不一样，先少量试，不要一次囤。

5. 太厚的包被
南方家庭真的不一定用得上，还容易热。

6. 网红收纳架
看起来整齐，但实际频繁拿取时很麻烦。

7. 功能重复的小电器
恒温壶、温奶器、消毒柜要看自己家动线，不要看别人买什么就跟着买。

8. 尺码囤太多的衣服
宝宝长得很快，有些衣服还没穿就小了。

我的建议是：先买基础款，少量试用，再根据宝宝和家庭习惯补。
```

### 3.3 预期反馈

应该出现：

```text
收藏：清单型内容有复看价值
追问：具体买什么型号？哪些不要买？
质疑：是不是广告？有没有品牌利益？
共鸣：我也踩过这个坑
专业风险：育儿内容不能说太绝对
```

### 3.4 不应出现

```text
大量用户都无脑点赞
所有评论都很礼貌正式
报告说预测真实点击率
Agent 像专家一样分析内容结构
```

## 4. Demo 2：装修避坑内容

### 4.1 基本信息

```json
{
  "title": "装修前一定要问清楚这 6 个问题，不然后期全是增项",
  "coverImage": "assets/demo/cover_renovation_tips.png",
  "scale": "standard"
}
```

### 4.2 正文

```text
装修最怕的不是一开始报价高，而是一开始看着便宜，后面不断增项。

签合同前建议至少问清楚 6 个问题：

1. 水电是按点位、按米，还是一口价？
2. 柜体报价是否包含五金、拉手、见光板？
3. 瓷砖铺贴是否包含找平、倒角、海棠角？
4. 乳胶漆是否包含基层处理和防裂处理？
5. 垃圾清运、成品保护、上楼费是否单独收费？
6. 后期变更项目如何计价，是否有书面确认？

不要只看总价，要看报价单里有没有漏项。

尤其是"按实际发生结算"的项目，一定要提前问清楚计算方式。
```

### 4.3 预期反馈

应该出现：

```text
收藏：装修前要对照问
追问：有没有报价单模板？
专业补充：不同城市计价方式不同
质疑：说得对但太笼统
风险：过度焦虑化标题
```

## 5. Demo 3：普通种草内容

### 5.1 基本信息

```json
{
  "title": "用了一个月，我终于理解为什么大家都推荐这个通勤包",
  "coverImage": "assets/demo/cover_general_recommendation.png",
  "scale": "standard"
}
```

### 5.2 正文

```text
之前一直觉得通勤包都差不多，直到这个包用了一个月，才发现细节真的影响体验。

我最喜欢的是这几点：

1. 自重比较轻，背电脑也不会特别累。
2. 分区合理，电脑、充电器、纸巾、钥匙都有位置。
3. 外形不夸张，上班和周末都能背。
4. 肩带不勒，通勤时间长也舒服。

但它也不是完美的：

1. 下雨天最好别直接淋。
2. 装太满会有点鼓。
3. 如果你经常带很多文件，可能不够大。

适合每天通勤、东西不算特别多、喜欢简洁风格的人。
```

### 5.3 预期反馈

应该出现：

```text
广告感质疑更强
追问品牌、价格、链接
对比需求：和某些品牌比怎么样？
收藏弱于育儿/装修清单
```

## 6. Demo 运行配置

建议：

```text
开发调试：quick，12 位观众
验收演示：standard，30 位观众
并发：5
每个 journey 最大 10 个 JourneyAction
每个 JourneyAction tool call 安全上限：20
```

## 7. Demo 验收观察点

### 育儿 Demo

重点看：

```text
是否出现广告质疑
是否出现具体型号追问
是否出现收藏行为
是否出现专业风险提醒
```

### 装修 Demo

重点看：

```text
是否出现模板追问
是否出现专业补充
是否出现收藏动机
是否出现合同 / 报价单相关评论
```

### 种草 Demo

重点看：

```text
是否出现广告感更强
是否追问价格品牌
是否收藏率较弱
是否有路人看到封面和标题后直接结束浏览
```

## 8. Prompt 对比方法

每次修改 prompt 后，用同一 Demo 跑：

```text
1. 记录点开、收藏、评论、信息流直接结束浏览数量。
2. 抽看 10 条 thought_text / action log。
3. 抽看 10 条评论。
4. 对比是否更像真实用户。
5. 记录异常行为。
```

## 9. 固定坏例

如果 Demo 跑出以下结果，需要调整 Prompt 或工具约束：

```text
1. 30 个观众几乎全部点赞收藏。
2. 评论都像用户访谈记录。
3. 没有人质疑广告感。
4. 所有人都按固定流程互动。
5. Agent 在 feed_card 就写评论。
6. 报告输出真实预测率。
7. 修改建议没有证据。
```
